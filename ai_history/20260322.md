## Major Achievements

### 1. ✅ Corrected Data Format Understanding

**Issue**: The device only outputs **acceleration** and **angular velocity** data (~8 columns), not gravity vectors.

**Resolution**: 
- Updated data mapping from 11 columns to 8 columns
- Changed field names: `ax,ay,az,gx,gy,gz,wx,wy,wz` → `accX,accY,accZ,gyroX,gyroY,gyroZ`
- Removed gravity-dependent code

### 2. ✅ Rewrote PostureAdjustment Algorithm

**Previous Approach**: Relied on explicit gravity vector from IMU

**New Approach** (Using Low-Pass Filtering):
- Estimate gravity vector from raw acceleration data using low-pass filtering
- Filter coefficient: α = 0.02 for smooth gravity estimation
- Filters out dynamic motion peaks while retaining constant gravitational component

**Algorithm Flow**:
```
Raw Acceleration (accX, accY, accZ)
         ↓
   Low-Pass Filter
         ↓
Estimated Gravity Vector
         ↓
Up Direction (opposite of gravity)
         ↓
Project acceleration to:
  - A-Forward (athlete's running direction)
  - A-Vertical (up/down)
  - A-Side (left/right, ~= 0)
```

### 3. ✅ Updated Front-End UI Table

**Column Order** (NEW):
```
Time | Milliseconds | A-Forward | A-Vertical | A-Side | Acc-X | Acc-Y | Acc-Z | Gyro-X | Gyro-Y | Gyro-Z
```

- Reduced from 14 columns to 11 columns
- Transformed results appear first (better UX)
- Shows both raw and converted data

### 4. ✅ Added Local File Loading Feature

**New Button**: "Load" - allows direct loading of CSV files from local filesystem
- No USB device connection required
- Same data parsing pipeline as device sync
- Useful for offline analysis and testing

### 5. ✅ Implemented Auto-Detection for Data Units

**Problem Identification**: Device outputs data in scaled units (e.g., gravity magnitude ≈ 51.2 instead of 1.0)

**Solution**:
- Auto-detect `gravityScaleFactor` on first data row
- Scale factor = Expected(1.0) / Measured(51.2) ≈ 0.0195
- Display scale factor in conversion status message
- Enables future unit conversion if needed

## Technical Details

### Low-Pass Filter Implementation
```javascript
// Exponential moving average
filteredAccel = α * current + (1-α) * previous
// α = 0.02 → uses ~4% current, 96% historical

// Separates:
//   - Long-term gravity (constant ~1g downward)
//   - Short-term motion (dynamic accelerations)
```

### Coordinate System Establishment
1. **Vertical (Up)**: Inverse direction of filtered acceleration
2. **Forward (Initial)**: Along X-axis by default (configurable)
3. **Right**: Cross product of Up × Forward
4. **Verification**: Ensure orthogonality via Re-orthogonalization

### Auto-Scale Detection
```javascript
EXPECTED_GRAVITY_MAG = 1.0  // Expected standard unit
measured = magnitude(acc)    // e.g., 51.2
scaleFactor = 1.0 / 51.2    // ≈ 0.0195
console.log(`Scale: ${scaleFactor}`) // Logged to browser console
```
